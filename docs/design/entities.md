# §10 Entity Catalog

| Entity | Interfaces | Description |
|---|---|---|
| `PlayerBlock` | `IWorldProp` | Prototype controllable player entity with input-driven movement, map-bounds clamping, and sprite-animated rendering. Y-sorted via `layerDepth`. |
| `FollowerBlock` | `IWorldProp` | Sidekick companion that samples the leader's recent path and stays a fixed distance behind it, preserving corners without offset steering. Y-sorted via `layerDepth`. |
| `Boulder` | `IWorldProp` | Static prop entity drawn from a sprite and used as a solid world obstacle. Y-sorted via `layerDepth`. Also used as a generic backing entity for decorative props without dedicated collision logic (e.g., seaweed, sunken logs, dock legs). |
| `Tree` | `IWorldProp` | Static world prop for tree entities. Each tree species defines **one or more** collision rectangles (e.g., pine trunk shape uses 4 stacked rectangles that hug the narrow trunk and wider root base) via a `localCollisionBoxes` array constructor parameter. Collision is defined once per species in `PropFactory` constants rather than hand-placed in the tilemap. Separates visual `Bounds` (full sprite, Y-sorting/occlusion) from per-index `GetCollisionBounds(i)` (trunk base, movement blocking). Y-sorted via `layerDepth`. |
| `Cabin` | `IWorldProp` | Static world prop for cabin/building entities. Same multi-box collision pattern as `Tree` — accepts a `localCollisionBoxes` array relative to the sprite origin, each rectangle covering a portion of the building footprint. `CollisionBoxCount` and `GetCollisionBounds(i)` expose all boxes to callers. Y-sorted via `layerDepth`. |
| `FrontDoor` | `IWorldProp` | Decorative front-door prop that selects authored open/closed sprites from TMX `propType` metadata and automatically opens only when the player is extremely close. Y-sorted via `layerDepth` and optionally suppresses occlusion reveal. |
| `Dock` | `IWorldProp` | Static decorative prop entity drawn from a sprite and placed in world space by gameplay screen data. Not Y-sorted (always draws behind sorted entities). |
| `SunkenChest` | `IWorldProp` | Static decorative prop entity drawn from a sprite and placed from TMX object-layer prop metadata, including underwater placement support. Y-sorted via `layerDepth`. |
| `Firepit` | `IWorldProp` | Interactive prop entity drawn from a sprite and used as a solid world obstacle, placed via TMX object-layer data. Can compose a `SmallFire`, expose nearby interaction bounds, and toggle its attached fire on/off. Y-sorted via `layerDepth`; passes depth to attached `SmallFire`. |
| `SmallFire` | `IWorldProp` | Animated visual effect entity drawn from a looping horizontal sprite sheet. Composes a `LoopAnimator` component for frame cycling, accepts externally attached smoke/spark particle emitters, and exposes a per-frame warm point-light snapshot for the lighting pass. Purely visual with no collision. Receives `layerDepth` from parent `Firepit`. |
| `GnomeEnemy` | `IWorldProp` | Lightweight enemy entity that hops toward a target position using a sine-wave vertical offset. Accepts an `EnemyType` property (set at spawn time) that parameterizes visual and behavioral traits — tint color, draw scale, base speed multiplier, HP, and optional on-death effects. All variants share the same sprite asset and state machine (chase → wind-up → lunge → stun → dying); the type only modulates numeric parameters and rendering. Drawn from a single sprite with flip-based facing. Y-sorted via `layerDepth`. |
| `MomNpc` | `IWorldProp` | Non-hostile NPC that patrols using an `IndoorNavigator` component backed by an `IndoorNavGraph`. Delegates route selection, progress tracking, and repath logic to `IndoorNavigator`; handles movement, foot-bounds collision, facing, and rendering locally. Drawn via `SpriteAnimator` (same 4×4 character sheet layout as player/follower). Y-sorted via `layerDepth`. |
| `Projectile` | `IWorldProp` | Pooled projectile entity that flies in a straight line at constant speed with a fixed lifetime and a small enemy pierce budget. Deactivates on obstacle hit, after consuming its pierce count, or on expiry. Drawn from a directional arrow sprite rotated from its velocity. Y-sorted via `layerDepth`. |
| `FishSilhouette` | — | Animated fish silhouette that swims in the underwater area of the fishing mini-game screen. Has a 7-state behavioral state machine (Unaware → Curious → Approaching → StrikeReady → Spooked → Hooked → Fleeing) that mimics natural fish attraction, response to the lure, and fright behavior. Draws from a 6-frame × 3-row sprite atlas (34×22px per cell). |
| `FlatShoreDepthSimulator` | `IWorldProp` | Purely visual underwater prop that simulates shore depth using a 1×3 tile sprite. Placed in the fishing scene to create a depth illusion at the waterline. No collision. Y-sorted via `layerDepth`. |
| `GardenGnome` | `IWorldProp` | Decorative gnome prop that peeks out from behind a tree. Slides behind its tree anchor and hides when the player approaches within a 3-tile proximity radius (96px), then slides back out when the player moves away. Drawn from an optional texture; logic-only constructor supported for tests. Y-sorted via `layerDepth`. |
| `HealthPickup` | `IWorldProp` | Pooled health pickup (pill) that auto-despawns after 10 seconds, fading linearly from fully opaque at 7s to invisible at 10s. Collectible within a 16px radius. Activated/deactivated by `HealthPickupSystem`. Drawing is handled externally by the system (entity does not store a texture). |
| `PropFactory` | — | Static factory that creates world prop entity arrays from TMX object-layer prop placements. Defines per-species collision box constants (e.g., `PineTreeCollisionBoxes`, `BirchTreeCollisionBoxes`, `CozyCabinCollisionBoxes`), and provides `CreateTrees()`, `CreateCabins()`, `GetTreeCollisionBounds()`, and `GetCabinCollisionBounds()` helpers for `GameplayScreen.LoadContent()`. |
| `IWorldProp` | — | Common interface for all world-placed prop entities that participate in Y-sorting and the world render pass. Requires `Vector2 Position`, `Rectangle Bounds`, and `void Draw(SpriteBatch, float layerDepth)`. Provides a default `bool SuppressOcclusion => false` for entities that should not activate the occlusion reveal lens. |
